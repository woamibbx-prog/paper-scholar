# Paper Scholar Skill 变更日志

## [2.1.0] - 2026-04-17

### 漏洞修复

本次更新修复了在 cron 循环任务机制中发现的 6 个漏洞，大幅提升了 skill 的稳定性和可靠性。

#### 1. 修复 kb.json 并发写入冲突（P0 - 严重）

**问题：** `_write_kb()` 使用 `write_text()` 写入文件，不是原子操作，可能导致并发写入时 kb.json 损坏。

**修复：**
- 改用临时文件 + `os.replace()` 的原子化写入方式
- 确保多任务并发时 kb.json 不会损坏
- 保留原有的备份机制

**影响：** 消除了多任务并发时 kb.json 损坏的风险，保障数据完整性。

#### 2. 添加任务状态跟踪（P2 - 轻微）

**问题：** 无法知道当前是否有任务正在运行，Cron 触发时无法检查任务状态。

**修复：**
- 添加 `.task_status.json` 文件记录任务状态
- 提供 `set_task_status()`, `get_task_status()`, `is_task_running()`, `clear_task_status()` 函数
- 自动检测僵尸任务（2 小时未更新）

**影响：** 可以追踪任务运行状态，避免并发触发。

#### 3. 增强 parse_part4() 的错误处理（P1 - 中等）

**问题：** 如果反思文件的 PART 4 格式不正确，会抛出异常，没有降级策略。

**修复：**
- 添加安全模式（safe_mode）
- 解析失败时返回安全的默认结果（继续当前领域）
- 提供错误信息和原始文本供参考

**影响：** 即使解析失败，任务也能继续执行，不会中断学习循环。

#### 4. 增强 load_kb() 的容错性（P1 - 中等）

**问题：** 如果 kb.json 损坏，`json.loads()` 会崩溃，没有从备份恢复的机制。

**修复：**
- 添加 JSON 完整性验证（`_validate_kb_structure()`）
- 实现从备份自动恢复（`_restore_from_backup()`）
- 损坏时自动从最新备份恢复

**影响：** 即使 kb.json 损坏，也能自动从备份恢复，不会导致任务崩溃。

#### 5. 增强 write_reflection() 的安全性（P2 - 轻微）

**问题：** 理论上两个任务可能同时写入同一个时间戳的文件（概率很低）。

**修复：**
- 确保时间戳唯一（如有冲突，添加后缀）
- 先检查文件是否存在再写入

**影响：** 消除了反思文件冲突的可能性（即使概率很低）。

#### 6. 修复 Cron 触发时机漏洞（P0 - 严重）

**问题：** Cron 每 20 分钟触发一次，不管上一个任务是否完成，可能导致多个任务并发运行。

**修复：**
- 在文档中明确说明需要配置 `maxConcurrent: 1`
- 添加任务状态跟踪，可以检测并发问题
- 提供手动限制并发的建议

**影响：** 通过 Cron 配置限制并发，避免多任务同时运行。

### 新增功能：自修复模块

本次更新新增了一个自修复模块 `self-repair.md`，让 skill 具备自我检查和自我修复的能力。

#### 功能概述

自修复模块在每轮任务执行完成后自动执行，负责：
1. **检查执行结果**：数据完整性、文件格式、执行结果
2. **检测漏洞**：并发冲突、解析失败、文件损坏等
3. **自动修复**：清理垃圾数据、修复数据不一致、更新 skill 文件
4. **生成报告**：保存修复报告供参考

#### 调用方式

在 learning-reflection 模块的最后添加调用：
```python
# 更新 learning_context
lib.kb.update_learning_context(kb, context)

# 🔴 新增：调用自修复模块
lib.self_repair.run_self_repair(kb, batch_id, summary_file)

# 清理任务状态
lib.kb.clear_task_status()
```

#### 修复报告示例

```json
{
  "timestamp": "2026-04-17T14:30:00",
  "batch_id": "2026-04-17_14-30-00",
  "checks": {
    "consistency": [...],
    "reflection_format": [...],
    "execution": [...],
    "vulnerabilities": [...]
  },
  "fixes": {
    "cleanup": {...},
    "kb_repair": {...},
    "skill_update": {...}
  },
  "issues": [...]
}
```

#### 修复原则

- **安全第一**：只修复有把握的问题，不确定的问题记录日志
- **保留备份**：修复前自动备份原始文件
- **透明记录**：所有修复操作都记录到修复日志
- **用户通知**：重要修复通知用户，可能需要确认

**问题：** 标准化规范分散在多个文档中（SKILL.md、STANDARDIZATION_GUIDE.md、各模块 .md 文件），导致容易遗漏或产生冲突。

**代码变更：**
```python
# 返回结果新增字段
result = {
    "action": "continue",
    "safe_mode": True,  # 是否为安全模式
    "error": "解析失败: ..."
}
```

#### 4. 增强 load_kb() 的容错性（P1 - 中等）

**问题：** 如果 kb.json 损坏，`json.loads()` 会崩溃，没有从备份恢复的机制。

**修复：**
- 添加 JSON 完整性验证（`_validate_kb_structure()`）
- 实现从备份自动恢复（`_restore_from_backup()`）
- 损坏时自动从最新备份恢复

**影响：** 即使 kb.json 损坏，也能自动从备份恢复，不会导致任务崩溃。

#### 5. 增强 write_reflection() 的安全性（P2 - 轻微）

**问题：** 理论上两个任务可能同时写入同一个时间戳的文件（概率很低）。

**修复：**
- 确保时间戳唯一（如有冲突，添加后缀）
- 先检查文件是否存在再写入

**影响：** 消除了反思文件冲突的可能性（即使概率很低）。

#### 6. 修复 Cron 触发时机漏洞（P0 - 严重）

**问题：** Cron 每 20 分钟触发一次，不管上一个任务是否完成，可能导致多个任务并发运行。

**修复：**
- 在文档中明确说明需要配置 `maxConcurrent: 1`
- 添加任务状态跟踪，可以检测并发问题
- 提供手动限制并发的建议

**影响：** 通过 Cron 配置限制并发，避免多任务同时运行。

### 新增功能：自修复模块

本次更新新增了一个自修复模块 `self-repair.md`，让 skill 具备自我检查和自我修复的能力。

#### 功能概述

自修复模块在每轮任务执行完成后自动执行，负责：
1. **检查执行结果**：数据完整性、文件格式、执行结果
2. **检测漏洞**：并发冲突、解析失败、文件损坏等
3. **自动修复**：清理垃圾数据、修复数据不一致、更新 skill 文件
4. **生成报告**：保存修复报告供参考

#### 检查项列表

1. **数据完整性检查**：
   - kb.json 与实际文件系统的一致性
   - 反思文件的格式正确性
   - 批次记录的完整性

2. **执行结果检查**：
   - 论文下载成功率
   - PDF 文件完整性
   - 总结文件完整性

3. **漏洞检测**：
   - 并发冲突检测
   - 解析失败检测
   - 文件损坏检测

4. **自动修复**：
   - 清理 0 字节 PDF
   - 清理临时文件
   - 清理空目录
   - 修复 kb.json 与文件系统的不一致

#### 调用方式

在 learning-reflection 模块的最后添加调用：
```python
# 更新 learning_context
lib.kb.update_learning_context(kb, context)

# 🔴 新增：调用自修复模块
lib.self_repair.run_self_repair(kb, batch_id, summary_file)

# 清理任务状态
lib.kb.clear_task_status()
```


#### 修复原则

- **安全第一**：只修复有把握的问题，不确定的问题记录日志
- **保留备份**：修复前自动备份原始文件
- **透明记录**：所有修复操作都记录到修复日志
- **用户通知**：重要修复通知用户，可能需要确认
